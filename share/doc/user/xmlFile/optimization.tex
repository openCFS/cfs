\chapter{Optimization}
The optimization part allows the shape, topology and material optimization of special problem. It is not a general optimization tool for arbitrary problems.

Optimization is enabled by appending an \topelement{optimization} element after the \element{sequenceStep} element. Generally one has to add additional result types to the \element{storeResults} of the simulation problem.

This are the main elements for defining an optimization problem:
\begin{itemize}
  \item The optimization problem with cost function and constraints
  \item The optimizer which is the tool that actually solves the problem
  \item A concretion of the optimization problem
\end{itemize}

\subsubsection{Optimization Problem}
\label{sec:optimization_problem}
The type of problem to be optimized is called ``cost function'' or ``objective function''. Here we use the symbol $J$. It is defined as attribute \attribute{type} in the element \element{costFunction}. The following cost functions are defined:

\subsubsection*{compliance}
Only defined for static mechanics. Makes only sense with minimization, hence we have
\begin{equation}
  J = {\bf u}^T {\bf f} = {\bf u}^T {\bf K_{uu}} {\bf u} \textrm{ as } {\bf K_{uu}}{\bf u} = {\bf f}.   
\end{equation}
This can be interpreted as the minimization of the mechanical energy, the maximization of the stiffness or the minimization of the displacement at the point and in the location an external force is applied. 

\subsubsection*{conjugateCompliance}
This is the time harmonic variant of compliance. It describes the problem
\begin{equation}
  J = {\bf \bar{u}}^T {\bf \bar{S}_{uu}}{\bf \bar{u}}^*,
\end{equation}
where all elements are complex, ${\bf \bar{S}_{uu}}$ is a combination of stiffness, damping and mass matrix and ${\bf \bar{u}}^*$ is the conjugate complex displacement vector of the global solution.

\subsubsection*{globalDynamicCompliance}
This is again a harmonic, mechanical based objective function. It is given as 
\begin{equation}
  J =  {\bf \bar{u}}^T {\bf \bar{u}}^*.
\end{equation}
This is a pure geometric objective function describing the total deformation.

\subsubsection*{tracking}
This is a static mechanic objective function. It makes only sense with minimization, the problem is given as 
\begin{equation}
  min = \frac{1}{2} \| {\bf u} - {\bf u}_* \|^2,
\end{equation}
where ${\bf u}_*$ is a predefined deformation. The problem is to minimize the least square error of the actual displacement ${\bf u}$ to the predefined one. Common is the notation ${\bf u}^*$ instead of ${\bf u}_*$ but this could create confusion with the conjugate complex. ${\bf u}_*$ is defined within the element \element{tracking}, a subelement of \subelement{ersatzMaterial}.

\subsubsection*{output}
This static objective function for mechanic, acoustic and piezoeletricity allows the optimization
of arbitrary parts of the solution (displacement and potential). Typically one maximizes here but also a minimization can make sense. This objective is also called ``synthesis of compliant mechanisms'' by Bends{\o}e and Sigmund as it alows to optimize for force invertes, grippers or simply maximal displacement of actors. The function is given as
\begin{equation}
  J = {\bf \hat{u}}^T {\bf l},
\end{equation}
where $\bf l$ is vector multiplies with the global solution vector $\bf \hat{u}$. In the pure mechanical case $\bf \hat{u}$ is the global displacement vector $\bf u$, in the piezoelectric case $\bf \hat{u}$ consists of mechanical displacements and electric potential $\bm \phi$. The relevant nodes are set within the \element{output} element within the \element{costFunction} element. There might be multiple elements within \element{output}.

\subsubsection*{dynamicOutput}
This is the time harmonic variant of \element{output}. It is given as
\begin{equation}
  J = {\bf \hat{u}}^T {\bf L} {\bf \hat{u}}^*,
\end{equation} 
where $\bf L$ is a diagonal matrix with a trace corresponding to $\bf l$. All from \textbf{output} applies here, too. Furtheremore it is possible to optimize withi the acoustic domain for the acoustic potential $\bm \Psi$ or acoustic pressure $\bf p$ as part of $\bf \hat{u}$.

\subsubsection*{volume}
Makes sense only for the SIMP case:
\begin{equation}
  J = \frac{1}{V_*}\sum_e^{NE} \rho_e V_e.
\end{equation}
Here $NE$ ar all design elements, $\rho_e$ is the pseudo density for a single
design element, $V_e$ is the volume fraction of this design element and $V_*$
is the total volume of all design elements if all were filled. Hence the
objective value is normalized with an within $\rho_min \ldots 1$.

\subsubsection{Factor}
\label{sec:factor}
For the objective function \textbf{dynamicOutput} only, one can define a factor, multiplying $\omega^2$ so that we get
\begin{equation}
  J = \omega^2 {\bf \hat{u}}^T {\bf L} {\bf \hat{u}}^*,
\end{equation} 
where $\omega$ is the frequency in $\textrm{s}^{-1}$. This corresonds to the time derivative of $\bf \hat{u}$, e.g. velocity instead of displacement.

\subsubsection{homogenizationTensor}
The design domain represents a periodic base cell and requires periodic boundary
conditions. The idea is, that the homogenized tensor $[c]^H$ of the optimized
base cell matches a given tensor $[c]^*$. This is based on the entries of the
elasticity tensor. 

Either all relevant entries are to be matched or only selected. This is done
with multiple \xml{constraint} elements llike the following:
\begin{verbatim}
<constraint name="homogenizationTensor" type="equal" value="-0.025  coord="12"/>
\end{verbatim}

It is also possible to supply a subelement \xml{tensor} of dimensions $3 \times3$ for the 2D case and $6 \times 6$ for 3D. Then the \xml{value} attribute
must not be given. The \xml{coord} value \xml{all} implicitely generates
4 constraints in 2D with the tensor coordinates \xml{11}, \xml{22}, \xml{112}
and \xml{33} and 9 constraints for 3D. This can be controlled in the info.xml
file.

\begin{verbatim}    
<constraint name="homogenizationTensor" type="equal" coord="all" >
  <tensor dim1="3" dim2="3">
    <real>
      0.05    -0.025  0.0
     -0.025  0.05    0.0
      0.0     0.0    0.01
    </real>
  </tensor>      
</constraint>
\end{verbatim}

It might be intersting to monitor the convergence of the constraints setting
\xml{deltaConstraints} in \xml{log} to \xml{true}. Also one could
monitor tensor entries not subject to a constraint via the \xml{mode}
\xml{observation}.

\subsubsection{homogenizationTracking}
This is the tracking function 
\begin{equation}
 || [c]^H-[c]^*||_2,
\end{equation}
which is the root of the sum of the squared tensor entry difference. It is
mandatory to have \xml{coord} set to \xml{all} and provice the \xml{tensor}
element.
\begin{verbatim}
<constraint name="homogenizationTracking" type="equal" coord="all">
  <tensor dim1="3" dim2="3">
    <real>
      0.05    -0.025  0.0
     -0.025  0.05    0.0
      0.0     0.0    0.01
    </real>
  </tensor>      
</constraint>
\end{verbatim}
