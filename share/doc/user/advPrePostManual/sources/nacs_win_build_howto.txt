                                 1. Ueberblick

Die Buildumgebung von NACS besteht unter Windows aus einer Ansammlung von
Werkzeugen und Bibliotheken, die in einer Verzeichnisstruktur auf einem
Windows Laufwerk liegen.
Es existieren Umgebungen mit Visual C++ 2003 und 2005 Werkzeugen.
Der Hauptcompiler unter Windows ist der Intel C++ 10.0. Es stehen aber auch die
nativen Microsoft Compiler zur Verfuegung.
Verwenden laesst sich die Umgebung auf zwei Arten:

- als Laufwerk in einer virtuellen Maschine (Virtual Box), z.B. unter Linux.

- oder als gemountetes Laufwerk in einer nativen Windows Umgebung, was die
  Kompilierzeit im allgemeinen verkuerzt

Zum Starten der Umgebungen stehen zwei BATCH Skripte mit entsprechenden
Verknuepfung zur Verfuegung:

- Visual C++ 2003 Umgebung: 
  + Visual Studio 2003 Toolkit & Intel 10.0 Shell
  + nacsenv_vs2003.bat
  + hellblaue Shell

- Visual C++ 2005 Umgebung: gelbe Shell
  + Visual Studio 2005 & Intel 10.0 Shell
  + nacsenv_vs2005.bat
  + hellgelbe Shell

In beiden Shells sind schon entsprechende Umgebungsvariablen gesetzt, die es
sofort ermoeglichen NACS ueber CMake zu erstellen und auch Tools wie EMacs,
TCL, Perl, LaTeX, etc. sind installiert.

================================================================================

                         2. Verwendung mit Virtual Box

Zur Verwendung mit Virtual Box ist es noetig Virtual Box zunaechst auf einem
Host Rechner zu installieren. Als virtuelle Festplatten muessen dann als
primaerer Master "Windows XP Professional.vmdk" und als primaerer Slave
"nacs_windows_dev.vmdk" installiert werden. In der Virtual Box befindet sich
die Build Umgebung dann auf Laufwerk D:. In der Virtual Box koennen die Build
Umgebung direkt von D:\Dev gestartet werden. Ansonsten ist das weitere Vorgehen
analog zum mounten der virtuellen Platte "nacs_windows_dev.vmdk" als natives
Laufwerk.
Es ist auch moeglich die virtuelle Platte "nacs_windows_dev.vmdk" zu einer
vorhandenen VMware Maschine hinzuzufuegen, da es sich bei dem Plattenabbild um
ein VMware Abbild handelt. Hier ist das weitere Vorgehen dann auch analog zum
mounten als native Platte.

                    3. Verwendung als gemountetes Laufwerk


Wenn man die Build Umgebung als gemountets Laufwerk in einer vorhandenen
Windowsinstallation nutzen moechte, muss man zunaechst das Hilfsprogramm
"vmware-mount" fuer Windows installieren (siehe "VMware-mount-5.5.0-18463.exe").
Damit ist es dann moeglich das Plattenabbild "nacs_windows_dev.vmdk" unter
einem vorgegebenen Laufwerksbuchstaben zu mounten.
Zum Beispiel mountet der Befehl

"c:\Programme\VMware\VMware DiskMount Utility\vmware-mount.exe" X: nacs_windows_dev.vmdk

den man in den Kommandointerpreter "cmd" eingibt, das Plattenabbild unter dem
Laufwerksbuchstaben X:. Zum loesen der Einbindung kann man schreiben:

"c:\Programme\VMware\VMware DiskMount Utility\vmware-mount.exe" X: /d [/f]

wobei /f ein gewaltsames Loesen zur Folge hat.

Ist das Laufwerk eingebunden, kann der Kompiliervorgang beginnen!

                                   4. Ablauf


Wir nehmen an, dass die Build Umgebung auf eine der beschriebenen Weisen
eingebunden worden ist, zum Beispiel unter K:. Nehmen wir weiter an, dass wir
die 2003er Umgebung verwenden wollen. Dann ist es noetig die Batchfiles und die
Verknuepfung auf ein lokales Laufwerk zu kopieren und die "DEVPATH" Variable im
Batchfile auf "K:\Dev" zu setzen. Starten laesst sich die Umgebung fuer Visual
C++ 2003 dann durch einen Klick auf den kopierten Link "Visual Studio 2003
Toolkit & Intel 10.0 Shell".
Entsprechendes gilt fuer die 2005er Umgebung. Es empfiehlt sich auf dem
jeweiligen Windowsrechner TortoiseSVN zu installieren. Falls das nicht schon 
geschehen ist, liegt unter k:\software ein Installationspaket. Alternativ lassen
sich in der Shell auch die von UNIX bekannten svn Kommandos ausfuehren. Nehmen
wir an, das unter "C:\NACS_TRUNK" der ausgecheckte Quellcode liegt und unter
"C:\NACS_BUILD" der Build liegen soll. In der Shell starten wir dann die
grafische Oberflaeche von CMake"cmakesetup" und tippen die entsprechenden Pfade
in die Eingabefelder am oberen Rand und klicken auf "Configure" und waehlen als
Projekttyp "NMake Makefiles". Dann klicken wir weiter auf "Configure" bis der
"Generate" Knopf benutzbar wird.
Nach einem Klick auf "Generate" werden die Makefiles in "C:\NACS_BUILD" erzeugt
und die GUI beendet sich. In der Shell wechseln wir dann nach "C:\NACS_BUILD"
und tippen "nmake". Der Buildprozess sollte dann beginnen und die bekannten
Ergebnisse liefern.

                                   4. Bemerkungen


Die Hauptcompiler sind der Intel C++ 10.0 (icl) und der Intel Fortran 10.0
(ifort). Auf diese Compiler sind die Umgebungsvariablen CC, CXX und FC gesetzt,
die CMake braucht. Will man den Microsoft C++ verwenden sollte man CC und CXX
auf "cl" setzen.

Neben dem "depends" (entspricht ldd unter UNIX) sind auch einige aber nicht
alle Werkzeuge von Visual Studio installiert. Als wichtigstes waeren hier
"nmake" und "vcbuild" zu nennen. Letzteres ermoeglicht die Erstellung von .sln
und .vcproj Projekten auf der Kommandozeile.

Zum betrachten und editieren von HDF5 Dateien ist der HDFViewer samt Java
Umgebung mit dabei. Um etwaige Virtuelle Maschinen nicht zu ueberfordern hat
die JavaVM Standardmaessig nur 32MB Speicher zur Vefuegung. Will man mehr
bereitstellen sollte man die Variable HDFVIEW_JVM_MEMORY in den Umgebungsbatch-
files auf einen hoeheren Wert stellen.
